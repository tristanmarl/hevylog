## Environment

Requires `.env` with:
```
VITE_HEVY_API_KEY=your-api-key-here
VITE_LIFTOSAUR_API_TOKEN=your-token-here   # enables live Liftosaur fetch via Vite proxy
```

Liftosaur data is fetched live via the Vite dev-server proxy (`/liftosaur-api` → `liftosaur.com/api/v1`) when `VITE_LIFTOSAUR_API_TOKEN` is set. Without it, falls back to `public/liftosaur-history.json` (generated by `node scripts/sync-liftosaur-history.mjs`).

## Architecture

Single-page React app (Vite + TypeScript + Tailwind) that visualises workout data from Hevy or Liftosaur.

**Types:** `src/types/workout.ts` is the canonical runtime shape (`Workout`, `WorkoutExercise`, `WorkoutSet`, `BodyweightEntry`). `src/types/hevy.ts` holds raw Hevy API response shapes; the API layer maps them to the canonical types.

**Data sources:** `src/api/dataSource.ts` is the single entry point for all data. It dispatches to `src/api/hevy.ts` (live REST API, 5-min TTL in-memory cache) or `src/api/liftosaur.ts` (parses `public/liftosaur-history.json`, sessionStorage cache). Liftosaur workouts are read-only. Active source (`hevy` | `liftosaur`) is persisted to `localStorage` via `DataVersionContext`.

**State:** `src/context/DataVersion.tsx` holds the active `DataSource` and a `version` counter. Calling `refresh()` clears the cache and bumps `version`, which causes pages to re-fetch. No other global state.

**Computation layer:** `src/utils/stats.ts` — all analytics (weekly stats, PRs, muscle balance, plateaus, streaks, milestones, suggestions) as pure functions over `Workout[]`. `src/utils/muscles.ts` infers muscle groups via keyword matching on exercise title, falling back to `ExerciseTemplate.primary_muscle_group`.

**Routing:** React Router v7, all routes under a single `Layout` shell with a collapsible sidebar and `⌘K` global search. Period selection (`week / 30d / 90d / 365d`) is persisted to `localStorage`.

**Styling:** Tailwind utility classes + occasional inline `style` props for brand orange `#e86a2e` and dark palette (`#0f0f0f` bg, `#1a1a1a` cards, `#2a2a2a` borders). No CSS modules or styled-components.
